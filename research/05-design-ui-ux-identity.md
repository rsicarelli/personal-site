FOCUS: Visual identity, UI/UX, design system, and look-and-feel for a senior engineer's personal site.

Research the current (2025–2026) landscape and best practices for the DESIGN of a credible,
distinctive personal developer/portfolio/blog site. Cover:
- Reference & inspiration: outstanding personal sites of senior/staff engineers, conference
  speakers, and technical writers — what makes them effective (cite real examples with URLs).
- Visual identity for a personal tech brand: color palette strategy, typography pairings
  (incl. good multilingual/diacritics support for pt-BR), spacing/layout systems, dark/light mode.
- UX/IA: how to structure navigation across blog, CV, portfolio, events, photos, materials, contact,
  and a future paid-content area, without it feeling cluttered.
- Design system / component approach: design tokens, theming, Tailwind vs CSS-in-JS vs vanilla/UnoCSS,
  headless UI/component libraries — surveyed neutrally for a content-first static-ish site.
- Accessibility (WCAG) and inclusive design as a baseline, not an afterthought.
- Showcasing assets well: talk-photo galleries, embedded talk videos (KotlinConf), OSS project
  cards, a polished CV view, and downloadable materials.
- Performance-aware design: fonts, images, motion that don't wreck Core Web Vitals.
- How to balance a recognizable personal brand with low long-term maintenance.

DELIVERABLES (in addition to the standard output contract): a few concrete moodboard-style
directions (described in words + referenced live examples), a suggested type+color system rationale,
and an IA/sitemap proposal — presented as options with trade-offs, not a single mandated design.
